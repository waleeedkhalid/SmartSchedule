# Notes & Schema Clarifications

- Student level 2 is invalid because only levels 4-8 are allowed (see schema: min 4, max 8).
- Student ID format is now YYYGXXXXX (Y=year, G=gender, X=digit, 9 chars).
- Valid slot days: Sunday to Thursday only. Friday and Saturday are considered weekend/off and are invalid.

# Example of a Fully Valid Set

## Student Setup
studentNumber: 123100001
level: "4"

## Student Login
studentNumber: 123100001
password: pass123

## Student Feedback
courseCode: CS101
rating: 5
feedback: Great course! Learned a lot.

## Faculty Setup
facultyId: F12345
title: Dr.

## Time Slot
{
	day: "Monday",
	startTime: "09:00",
	endTime: "10:00"
}

# Invalid Examples
- Student level: 2 (invalid, must be 4-8)
- Student number: 12345 (invalid, must match YYYY-XXXXX)
- Time slot: day: "Friday" (invalid, weekend)

SignIn valid: { email: user@email.com, password: pass123 }
SignIn invalid: { email: bad, password: (empty) }

SignUp valid: { email: user@email.com, password: StrongPass1, confirmPassword: StrongPass1, fullName: John Doe, role: student }
SignUp invalid: { email: bad, password: weak, confirmPassword: notmatch, fullName: J1, role: unknown }
